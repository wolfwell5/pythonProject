## git 自动补全提交信息  
feat(stock): 添加股票数据获取和对比功能 这句话啥意思？ feat(stock)


feat(stock): 添加股票数据获取和对比功能       
│    │       └─ 具体描述         
│    └─────── 作用范围（scope）   
└──────────── 提交类型（type）    
 

各部分含义：
1. feat - 提交类型  
表示这是一个 新功能（feature）  
常见的类型有：   

| 类型       | 含义      | 示例              |
|----------|---------|-----------------|
| feat     | 新功能     | feat: 添加用户登录    |
| docs     | 文档更新    | docs: 更新 README |
| style    | 代码格式调整  | style: 格式化代码    |
| refactor | 重构代码    | refactor: 优化性能  |
| test     | 测试相关    | test: 添加单元测试    |  
| chore    | 构建/工具配置 | chore: 更新依赖     | 


2. (stock) - 作用范围（scope）
表示这个改动影响的是 stock 模块/目录
其他可能的范围：
(user) - 用户模块 
(api) - API 接口  
(ui) - 界面相关  
(config) - 配置文件 

3. 添加股票数据获取和对比功能 - 简短描述
具体说明做了什么改动
